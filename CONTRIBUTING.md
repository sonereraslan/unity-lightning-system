# Contributing

Thanks for your interest in contributing! We welcome all kinds of contributions: bug reports, feature requests, and code improvements.

## Getting Started

1. Fork the repository
2. Clone:  git clone https://github.com/sonereraslan/unity-lightning-system.git
3. Open with Unity (version: 6.3)
4. Install required packages (URP, Input System, etc.)

## Branching Strategy

- main → stable
- develop → active development
- feature/* → new features
- fix/* → bug fixes

## Coding Guidelines

- Use clean and readable code
- Follow C# naming conventions
- Avoid large monolithic classes
- Systems should be decoupled
- Avoid heavy logic inside MonoBehaviours

## Commit Messages

Use clear commit messages:

- feat: add player movement
- fix: resolve null reference in AI system
- refactor: simplify loader logic

## Pull Requests

1. Create a new branch
2. Make your changes
3. Test your code
4. Open a Pull Request
5. Describe what you changed and why

## Reporting Issues

Please include:

- Description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Screenshots (if applicable)

## Types of Contributions

- Bug fixes
- New features
- Documentation improvements
- Performance optimizations
